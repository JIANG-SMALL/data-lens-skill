# Data Intake — 数据导入与解析

## 支持的全部输入形式

用户的数据可能以**任何形式**出现，DataLens 需要全部覆盖。

### 一、文件型输入

| 格式 | 扩展名 | 解析方案 | 特殊处理 |
|------|--------|---------|---------|
| Excel | `.xlsx` `.xls` `.xlsm` | `xlsx` (SheetJS) | 多sheet=多实体表；合并单元格展开 |
| CSV/TSV | `.csv` `.tsv` | `papaparse` | 自动检测分隔符(`,` `\t` `;` `\|`)；编码检测(chardet) |
| JSON | `.json` | 原生 `JSON.parse` | 数组=多条记录；对象=单条嵌套记录；嵌套结构拍平 |
| PDF | `.pdf` | `pdf-parse` + 表格检测 | 提取所有文本→识别表格区域→重建行列结构；扫描版PDF需OCR |
| Word | `.docx` `.doc` | `mammoth` / `word-extractor` | 提取嵌入的表格；合并跨页表格 |
| SQL Dump | `.sql` | 正则提取 | 解析 CREATE TABLE + INSERT INTO |
| SQLite | `.sqlite` `.db` | `better-sqlite3` | 提取所有表和字段；自动重建表关系 |
| Markdown | `.md` | 正则匹配 \|...\| | 识别GFM表格语法；多表格分别处理 |
| HTML | `.html` `.htm` | `cheerio` / `jsdom` | 提取所有 `<table>` 元素；处理 colspan/rowspan |
| XML | `.xml` | `fast-xml-parser` | 提取重复节点为行，属性/子节点为列 |
| 文本表格 | `.txt` | 启发式检测 | 尝试空格对齐/制表符/固定宽度分割 |

### 二、非文件型输入

| 形式 | 用户怎么提供 | AI 怎么处理 |
|------|------------|-----------|
| **📸 截屏图片** | 截屏/拍照后直接发到对话框 | OCR识别→表格检测→重建结构化数据 |
| **📋 直接粘贴** | Ctrl+V 粘贴表格内容 | 启发式检测分隔符→解析为表格 |
| **🌐 网页链接** | 发送一个URL | 抓取页面→提取所有 `<table>` →用户选择目标表格 |
| **📱 手机拍照** | 拍摄纸质报表/白板/屏幕 | OCR+透视矫正→表格检测→结构化 |
| **🔗 在线表格链接** | Google Sheets / 腾讯文档 / Airtable 链接 | 根据平台API获取数据或引导用户导出为CSV |
| **✍️ 手写笔记** | 拍摄手写表格 | OCR手写识别→结构化（准确率依赖笔迹清晰度，需标注置信度） |
| **🗄️ 数据库直连** | 连接字符串（如 mysql://...） | 引导用户先导出为CSV/SQL，或使用Cloud Functions直连（需用户授权） |
| **🔌 API端点** | 提供API URL | 通过Cloud Functions调用→获取JSON→解析 |

### 三、混合输入

用户可能同时提供多种形式的数据：
- 一个 Excel + 一个 PDF（两个文件分别解析后，AI 主动检测跨文件关系）
- 截图 + 粘贴文本（两张表可能来自同一上下文）
- 多个在线链接（不同数据源的关联分析）

## 🖼️ 图像/截图识别流程（重点能力）

这是最贴近用户实际场景的能力——很多人发来的就是一张截图。

```
用户发送图片
    ↓
1. 预检测：这张图里有没有表格？
   - 有 → 进入表格提取流程
   - 没有 → 告知用户"没检测到表格数据，请确认图片内容"
    ↓
2. OCR 识别（Tesseract.js 或云端OCR API）
   - 提取图片中所有文字及其位置坐标
    ↓
3. 表格结构重建
   - 检测规则排列的文字块 → 推断为表格
   - 检测直线/边框 → 确认行列边界
   - 合并跨行/跨列单元格
    ↓
4. 数据质量评估
   - 识别率: X%（太低则提示用户提供更清晰的图片）
   - 不确定的字符标注为 ⚠️
    ↓
5. 展示识别结果 → 让用户确认
   📸 图片识别结果
   ├── 来源: 截图/拍照
   ├── 检测到表格: 1个
   ├── 识别率: 94%
   ├── 行数: 230 × 列数: 9
   ├── 不确定单元格: 12个（已高亮标注）
   └── 请问识别是否正确？[确认] [我需要修正某些单元格]
```

### OCR 质量标注

对于识别不确定的单元格，向用户展示原图裁剪区域+识别结果：

```
⚠️ 第8行第3列识别不确定：
   [原图裁剪区域]
   识别结果: "12,5Q0" ← 看起来是 "12,500"
   [A] 改为 "12,500"  [B] 保留原结果  [C] 手动输入___
```

## 🔗 网页链接识别流程

```
用户发送URL → https://example.com/report
    ↓
1. 抓取页面HTML → 提取所有 <table> 标签
2. 如果页面有多个表格，列出让用户选：

   该页面检测到 3 个表格：
   表格1: "2024年度销售数据" — 50行 × 8列
   表格2: "客户分布统计"     — 12行 × 5列
   表格3: "月度汇总"         — 12行 × 4列

   请问要分析哪个？[全部分析] [只分析表格1] [只分析表格2] ...

3. 如果页面是动态加载（SPA/React渲染），告知用户：
   "该页面数据可能是动态渲染的，建议你截屏或导出为文件后上传"
```

## 解析后统一流程

所有输入形式解析完成后，统一进入以下流程（与原始输入形式无关）：

```
结构化数据 → 数据类型推断 → 数据画像 → 进入语义理解阶段
```

### 数据类型推断规则

| 样本特征 | 推断类型 | 示例 |
|---------|---------|------|
| 纯数字 + 无前导零 | number | `1024`, `3.14` |
| 数字+分隔符 + 长度>6 | ID/编码 | `2024-001-ABC` |
| YYYY-MM-DD / YYYY/MM/DD | date | `2024-01-15` |
| 含 http/https/ftp | url | `https://...` |
| 含 @ 且含 . | email | `user@domain.com` |
| 纯中文/英文短语 | text/category | `医疗器械`, `Active` |
| 布尔标识 | boolean | `是/否`, `true/false`, `1/0` |
| 小数+±符号 | percentage/ratio | `12.5%`, `+3.2` |
| OCR不确定 | uncertain | `12,5Q0`（可能是`12,500`） |

### 数据画像输出

解析完成后必须输出数据画像：

```
📊 数据画像
├── 来源: sales_2024.xlsx + 截图_20240510.png（混合输入）
├── 输入形式: Excel文件 + 手机拍照
├── 总行数: 12,847（文件）+ 230（图片）
├── 实体表数: 4
│   ├── Sheet1 "订单明细" → 8,420行 × 14列
│   ├── Sheet2 "客户信息" → 3,200行 × 9列
│   ├── Sheet3 "产品目录" → 1,227行 × 7列
│   └── 图片表格1 "手写盘点表" → 230行 × 9列（识别率: 94%）
├── 潜在跨来源关系:
│   ├── Sheet1.产品编码 ↔ Sheet3.产品编码
│   └── Sheet1.产品名称 ↔ 图片表格1.物品名称（相似度: 0.82, 待确认）
├── 数据质量:
│   ├── Sheet1.备注 (23.4% 缺失)
│   ├── Sheet2.联系电话 (11.2% 缺失)
│   └── 图片表格1 (6% 单元格识别不确定)
└── 需确认的不明确词汇: 5个
```

## 大文件/大图处理

- 文件：前端预览前500行；完整解析通过 Cloud Functions `/api/parse` 异步
- 图片：超过 10MB 自动压缩后再OCR；超大图片分区域分块识别
- 进度反馈：WebSocket 推送解析进度百分比

## 词汇澄清交互

对于每个不明确词汇（包括OCR不确定的单元格），向用户提出结构化问题：

```
🤔 检测到不明确词汇: Sheet1 列 "ZLX"

该列样本值: "ZLX-A01", "ZLX-B12", "ZLX-C03"

请确认:
[A] "诊疗类型" 的分类编码 → 分类维度
[B] "质量类型" 的分类编码 → 分类维度
[C] "治疗路线" 的分类编码 → 分类维度
[D] 直接输入含义_________________

该列是否作为分析的分类维度? [是] [否]
该列是否与其他列存在引用关系? [是 → 请指定目标列] [否]
```

用户的每个回答都记录到数据模型中，作为后续分析的语义基础。
